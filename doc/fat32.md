# FAT32 文件系统

TEST-OS 在 IDE 第二块盘（`FAT-FS.img`）上实现 FAT32，遵循 Microsoft FAT32 规范。

## 使用

1. 启动系统后格式化数据盘：

```
disk -f fat32 a:
```

2. 切换路径并操作文件：

```
cd a:
md docs
cd docs
pushf "Hello FAT32" > hello.txt
popf -a hello.txt
```

## 路径约定

- 路径以 **2 字节盘符** 开头：`a:` = `0x3a61`
- 盘符后接 `/` 与 Unix 风格路径：`a:/dir/file.txt`
- 当前工作目录由 eval 维护，可通过 `cd` 切换

## 已实现功能

| 功能 | 状态 |
|------|------|
| 检测 / 格式化 | 支持 |
| 打开 / 关闭文件 | 支持 |
| 目录遍历 | 支持 |
| 创建文件 / 目录 | 支持 |
| 删除 / 重命名 | 支持 |
| 读取任意偏移 | 支持 |
| **写入与扩容** | 支持（见下文） |
| 根目录扩容 | 不支持（根目录簇链固定） |
| 长文件名 (LFN) | 不支持（仅 8.3 短名） |
| FSInfo 空闲簇计数同步 | 部分（分配时更新 FAT，未写回 FSInfo） |

## 文件写入与扩容

`fat32_file_write()` 在 `offset + length` 超出当前文件大小时，会调用 `fat32_extend_file()`：

1. 按目标大小计算所需簇数
2. 在 FAT 中搜索空闲簇（值 `0`）
3. 将新簇链接到文件簇链末尾；空文件则分配首个簇并更新目录项 `first_cluster`
4. 新簇清零后写入数据
5. 更新目录项中的 `size` 与 `first_cluster`

### 支持的场景

- 向 **空文件**（`first_cluster = 0`，`size = 0`）首次写入
- 在文件末尾 **追加**（`pushf ... >> file`）
- 写入跨越 **簇边界** 导致文件变大
- 在 **子目录** 与 **根目录** 下更新目录项（根目录通过 `rootdir_cluster` 簇链写入）

### 限制

- 根目录本身不能通过 `md` 无限扩展（目录满时返回错误）
- 子目录可通过扩展目录文件的簇链添加新目录项
- 截断文件（缩小 size）未实现
- 无文件锁；并发写入同一文件不安全

## 内部结构（开发者）

```
disk (抽象层)
  └── fat32_fs (BPB、首数据簇等)
        ├── fat32_file (blocklist + dir_entry)
        └── fat32_dir  (目录项缓存)
```

关键函数：

| 函数 | 作用 |
|------|------|
| `fat32_extend_file` | 分配簇链并更新大小 |
| `fat32_file_rw` | 按簇读写数据 |
| `fat32_dir_update_entry` | 写回目录项（根目录与子目录统一路径） |
| `commit_fat32_buffer` | 将 FAT 缓存写回磁盘 |
